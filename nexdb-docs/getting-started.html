Getting Started

Everything you need to install NexDB, write your first application, and leverage its high-performance document capabilities in your language of choice.

Viewing examples in:

Installation

NexDB provides native libraries and package integrations for Rust, Python, and Node.js. Select your language to view installation steps.

install-dependency

Add nexdb-core to your Cargo.toml dependencies:

[dependencies]
nexdb-core = "0.1.0"
serde_json = "1.0"
tokio = { version = "1.0", features = ["full"] }

To enable optional binary BSON decoding features, compile with the bson flag:

nexdb-core = { version = "0.1.0", features = ["bson"] }

Install the Python client via pip:

pip install nexdb

Note: The Python client compiles the underlying Rust engine binary automatically during installation. Ensure you have the rustc toolchain installed on your machine.

Install the Node.js client package via npm:

npm install nexdb

Tip: The Node.js package downloads the appropriate pre-compiled high-performance Rust binary automatically based on your OS architecture during installation.

Quick Start

Learn how to open a database file, create a collection, insert a JSON document, and retrieve it.

quickstart-example 
use nexdb_core::{NexDb, Document};
use serde_json::json;

#[tokio::main]
async fn main() -> Result<(), Box> {
    // Open a database file (created automatically if missing)
    let db = NexDb::open("mydb.nex").await?;
    
    // Collections are created implicitly when accessed
    let collection = db.collection("users");
    
    // Insert a document with explicit ID
    collection.insert("u1", Document::from_value(json!({
        "name": "Alice",
        "age": 30,
        "city": "Berlin"
    }))).await?;
    
    // Retrieve the document
    let user_doc = collection.get("u1").await?;
    println!("Retrieved user: {}", user_doc.to_json());
    
    Ok(())
}
from nexdb import NexDb

# Open database file connection
db = NexDb('mydb.nex')
db.connect()

# Create collection handle
db.create_collection('users')

# Insert document with custom key
db.insert('users', 'u1', {
    'name': 'Alice',
    'age': 30,
    'city': 'Berlin'
})

# Retrieve document
user = db.get('users', 'u1')
print(f"Retrieved user: {user}")

db.close()
const { NexDb } = require('nexdb');

async function main() {
  // Initialize and connect to subprocess REPL
  const db = new NexDb('mydb.nex');
  await db.connect();

  // Create collection
  await db.createCollection('users');

  // Insert key-value document
  await db.insert('users', 'u1', {
    name: 'Alice',
    age: 30,
    city: 'Berlin'
  });

  // Get document by ID
  const user = await db.get('users', 'u1');
  console.log('Retrieved user:', user);

  await db.close();
}

main().catch(console.error);

You can also complete the same operations from your shell using the CLI tool:

# Create and insert
cargo nexdb insert mydb.nex users u1 '{"name":"Alice","age":30,"city":"Berlin"}'

# Get document
cargo nexdb get mydb.nex users u1

CRUD Operations

NexDB supports typical JSON collection methods to insert, query, update, delete, check existence, and count documents.

crud-reference

// Returns String ID (UUID v4)
let id = collection.insert_auto_id(Document::from_value(json!({
    "item": "canvas",
    "qty": 100
}))).await?;

Get Document

let doc = collection.get(&id).await?; // Returns NexDbResult

collection.update(&id, Document::from_value(json!({
    "item": "canvas",

    "updated": true

collection.delete(&id).await?;

let total = collection.count().await?; // Returns usize

let exists = collection.exists(&id).await?; // Returns bool

Query Builder

The query engine allows filtered document retrieval. You can query natively in Rust using a fluent query builder, or send filtering parameters over the TCP connection.

Note: Python, Node.js, and CLI client applications communicate with a running NexDB server instance via a TCP socket connection using a database connection string URL.

Here is how to run a query containing sorting, pagination limits, and operators in Rust:

let results: Vec<Document> = collection
    .query()
    .where_clause(r#"{"age": {"$gte": 21, "$lt": 65}, "city": "Berlin"}"#)?
    .sort("name", true) // Sort by name ascending
    .skip(10)          // Pagination skip
    .limit(20)         // Pagination limit
    .run().await?;

Indexes

B-Tree indexes accelerate retrieval speeds. When an index exists on a field, the engine utilizes the B-Tree structure to narrow documents in O(log n) time instead of performing a full collection scan running in O(n).

index-management

Create index on fields (supporting nested structures via dot notation):

// Index simple field
collection.create_index("age").await?;

// Index nested field
collection.create_index("address.zip").await?;

Create index by declaring a unique index handle and field path:

# create_index(collection, index_name, field_path)
db.create_index('users', 'age_idx', 'age')
db.create_index('users', 'zip_idx', 'address.zip')

Create index by declaring target collection, name, and field path:

// createIndex(collection, indexName, fieldPath)
await db.createIndex('users', 'age_idx', 'age');
await db.createIndex('users', 'zip_idx', 'address.zip');

WAL and Checkpoint

NexDB guarantees safety against sudden server failures by utilizing an append-only Write-Ahead Log. All database updates (inserts, updates, deletes) write to dbname.wal on disk before being applied in memory.

On startup (NexDb::open()), any entries present in the WAL that have not yet been snapshot-persisted are replayed automatically. To prevent WAL file size from growing indefinitely, call the checkpoint() operation periodically.

checkpoint-db

Write the database snapshot to disk and truncate the active WAL:

db.checkpoint().await?;

Flush WAL modifications to the underlying disk file:

db.flush()

Flush changes to the file system:

await db.flush();

TCP Server & Remote Hosting

Start a standalone NexDB database server. This mode lets you host the engine remotely (e.g. on panels like Pterodactyl or inside Docker containers) and connect to it from separate client projects using a connection string URL.

# Start the TCP server pointing to a databases directory
cargo run --manifest-path=nexdb-core/Cargo.toml -- serve ./data --port 27017 --metrics-port 28017

Connection URI Format

Remote applications connect using the standard NexDB connection URL format:

nexdb://[auth_token]@[server_ip_or_domain]:27017/[database_name]

Client Integration Examples

Choose your programming language to see how to establish a socket stream connection using the connection string:

connection-examples

Connect using the native Rust asynchronous client:

use nexdb::NexDbClient;
use serde_json::json;

#[tokio::main]
async fn main() -> Result<(), Box> {
    let mut client = NexDbClient::new("nexdb://token123@127.0.0.1:27017/prod_db")?;
    client.connect().await?;
    
    let res = client.query("insert", json!({
        "collection": "users",
        "id": "u1",
        "document": { "name": "Alice" }
    })).await?;
    
    println!("Response: {:?}", res);
    Ok(())
}

Connect using the Python client socket wrapper:

from nexdb import NexDbClient

db = NexDbClient('nexdb://token123@127.0.0.1:27017/prod_db')
db.connect()

db.insert('users', 'u1', {'name': 'Alice'})
user = db.get('users', 'u1')
print("Retrieved:", user)
db.close()

Connect using the Node.js client SDK with built-in connection pipelining:

const { NexDbClient } = require('nexdb-sdk');

async function run() {
  const db = new NexDbClient('nexdb://token123@127.0.0.1:27017/prod_db');
  await db.connect();
  
  await db.insert('users', 'u1', { name: 'Alice' });
  const user = await db.get('users', 'u1');
  console.log("Retrieved:", user);
  db.close();
}
run();

Prometheus Cluster Telemetry

NexDB servers expose cluster telemetry metrics in Prometheus format for integration with Grafana, Datadog, or cloud monitoring nodes. The exporter is served at the configured metrics port (default: 28017):

curl http://localhost:28017/metrics

Output values show: nexdb_active_connections, nexdb_reads_total, and nexdb_writes_total.

Import and Export

Import and export JSON or CSV datasets directly using CLI utilities.

# Import a JSON list of documents
cargo run --release -- import nexdb://secret@127.0.0.1:27017/prod_db users users_data.json

# Export a collection as a JSON list
cargo run --release -- export nexdb://secret@127.0.0.1:27017/prod_db users output_users.json

# Import from CSV (requires headers on the first line)
cargo run --release -- import-csv nexdb://secret@127.0.0.1:27017/prod_db users users_data.csv

# Export collection as CSV file
cargo run --release -- export-csv nexdb://secret@127.0.0.1:27017/prod_db users output_users.csv

Migration Engine

NexDB provides a built-in migration module to create database backups, restore collections, copy data between engines, or export collections into SQL syntax.

# Dump all database collections into JSON files
cargo run --release -- migrate dump nexdb://secret@127.0.0.1:27017/prod_db ./backup_dir/

# Restore all collections from JSON files
cargo run --release -- migrate restore nexdb://secret@127.0.0.1:27017/prod_db ./backup_dir/

# Copy all collection data directly between database instances
cargo run --release -- migrate copy nexdb://secret@127.0.0.1:27017/src_db nexdb://secret@127.0.0.1:27017/dst_db

# Output SQL commands (CREATE TABLE + INSERT) for another SQL dialect (pg/mysql/sqlite)
cargo run --release -- migrate to-sql nexdb://secret@127.0.0.1:27017/prod_db sqlite > dump.sql

# Import file automatically detecting format
cargo run --release -- migrate import nexdb://secret@127.0.0.1:27017/prod_db users source_file.json

You can also run database migrations programmatically inside your Rust application:

use nexdb::migrate;

// Dump all collections to backup folder
let manifest = migrate::dump(&mut client, "./backup_dir/").await?;
println!("Dumped {} docs", manifest.total_docs);

// Restore collections from folder
let restore_manifest = migrate::restore(&mut client, "./backup_dir/").await?;

Shell Completions

Generate native autocompletion scripts for shell environments to assist when using CLI commands.

# bash shell completion
cargo run --release -- completions bash > /etc/bash_completion.d/nexdb

# zsh shell completion
cargo run --release -- completions zsh > /usr/local/share/zsh/site-functions/_nexdb

# fish shell completion
cargo run --release -- completions fish > ~/.config/fish/completions/nexdb.fish

# powershell completion
cargo run --release -- completions powershell > $PROFILE

Clean Command

Wipe database files and WAL logs safely from your directories.

# Wipes target database files and wal locally on server host
cargo run --release -- clean ./data/mydb

# Wipes ALL .nexdb / .nex files and corresponding WAL logs under target folder
cargo run --release -- clean --all ./data/

BSON Support

When compiled with the bson cargo feature, NexDB supports inserting documents formatted in binary JSON (BSON). The engine parses BSON elements transparently into its internal JSON representation.

nexdb-core = { version = "0.1.0", features = ["bson"] }
